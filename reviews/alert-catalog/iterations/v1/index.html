A01 · Reservation

Missing reservation

Error

When it happens

An OT reaches its start without all required reels reserved.

Why the alert exists

Warehouse cannot reliably prepare the correct material.

Possible causes

The reservation was forgotten, incomplete, or created late.

Example

OT 151200.1 starts requiring four reels, but only three are reserved.

Detection indicators and algorithm

Compare the OT's required materials with its reservation records at the configured pre-start deadline and actual start. Alert when required reservations are missing.

A02 · Dispatch

Material not dispatched

Por vencer, then Error

When it happens

Reserved reels have not been sent before the required pre-start time.

Why the alert exists

The OT may start without material at the machine.

Possible causes

Warehouse delay, missing dispatch action, or incorrect schedule.

Example

The OT starts at 10:00 and its reels remain in the warehouse at 09:30.

Detection indicators and algorithm

Calculate planned start minus configured dispatch lead time. Show Por vencer before the cutoff and an error after the cutoff when any reserved reel lacks a dispatch event.

A03 · Transit and receipt

Material in transit too long

Error

When it happens

A dispatched reel has not been digitally received within the permitted interval.

Why the alert exists

The material's recorded location is uncertain.

Possible causes

Operator forgot receipt, physical movement was delayed, or the destination was incorrect.

Example

A reel remains En tránsito for 38 minutes against a 30-minute limit.

Detection indicators and algorithm

Find dispatched reels with no receipt event. Calculate current time minus dispatch time; alert when it exceeds the configured transit threshold.

A04 · Production sequence

OT started outside sequence

Error

When it happens

An operator starts an OT that is not next in the latest approved machine sequence.

Why the alert exists

Production is no longer following the supervisor's current plan.

Possible causes

Operator selected the wrong OT or used an outdated sequence.

Example

P12 starts OT 151104.1 while OT 151099.1 is first in the approved sequence.

Detection indicators and algorithm

At the actual start timestamp, compare the started OT with the first pending OT in the latest supervisor-approved sequence for that machine.

A05 · Consumption

Active OT without consumption

Error

When it happens

An OT remains active beyond the allowed initial interval without any consumption declaration.

Why the alert exists

Production cannot operate without raw material, so the digital workflow is incomplete.

Possible causes

Operator forgot to declare consumption or started the OT too early.

Example

An OT has been active for 15 minutes with zero consumed reels.

Detection indicators and algorithm

Require OT active, elapsed time since start greater than the configured interval, and consumption count equal to zero. Production declarations may support the same alert but must not create a duplicate.

A06 · OT closure

Declared meters exceed consumed-reel meters

Error

When it happens

At closure, declared run meters materially exceed the estimated meters provided by consumed reels.

Why the alert exists

The declared production cannot be explained by recorded consumption.

Possible causes

Missing consumption declaration, incorrect run meters, incorrect reel data, or incorrect closure.

Example

Consumed reels support approximately 30,000 m, but the operator declares 40,000 m.

Detection indicators and algorithm

Estimate meters in every consumed reel from weight, width, and basis weight. Sum them and compare them with declared run meters. Alert when the difference exceeds the configured tolerance. This is the primary closure rule.

A07 · OT closure

Completed OT with delivered reserved reels unconsumed

Error

When it happens

An OT completes all planned production, but a reserved reel delivered to the machine remains unconsumed.

Why the alert exists

If full production was completed, the delivered reserved material should have been used and declared.

Possible causes

Missing consumption declaration, incorrect completion status, or incorrect reservation quantity.

Example

Four reels were reserved and delivered; full production was completed, but only three were consumed.

Detection indicators and algorithm

Require full planned production completed, reserved reels delivered to the machine, and delivered reserved reels minus consumed reels is not empty. Do not apply automatically to truncated OTs.

A08 · OT closure

Truncated OT

Notice

When it happens

An OT closes before completing planned production or consuming all reserved material.

Why the alert exists

The incomplete production must be understood, but truncation is not automatically an error.

Possible causes

Machine failure, material problem, schedule change, or authorized early termination.

Example

An OT stops at 70% of the plan with one reserved reel unused.

Detection indicators and algorithm

Show a review notice when the OT has a truncation or partial-completion status. Include the recorded reason, requester, approval, and remaining material.

A09 · Production declaration

Possible undeclared produced reel

Possible error

When it happens

Input mass exceeds what could physically remain on the rewinder without additional produced-reel declarations.

Why the alert exists

One or more completed reels may have been removed without being declared.

Possible causes

Operator forgot the declaration, declared it late, or the mass assumptions are inaccurate.

Example

The rewinder holds at most 500 kg, but 1,500 kg has been loaded without enough output declarations.

Detection indicators and algorithm

Calculate unaccounted kg as input kg minus declared output kg minus declared waste kg minus allowed process loss. Warn when unaccounted kilograms exceed rewinder capacity plus tolerance. This is inferred because no PLC signal currently exists.

A10 · Pickup and movement

Produced reel not collected

Error

When it happens

A declared reel remains at the machine beyond the permitted pickup interval.

Why the alert exists

It can obstruct production and has not entered the weighing or downstream flow.

Possible causes

Process-team delay, missing movement record, or incorrect location.

Example

A reel declared at P15 remains recorded there after 30 minutes.

Detection indicators and algorithm

Start the timer at the production declaration. Alert when no pickup or movement event exists and the reel remains at the machine beyond the configured threshold.

A11 · Weighing

Produced reel not weighed

Por vencer, then Error

When it happens

A declared produced reel has no recorded weight within the permitted interval.

Why the alert exists

Its inventory quantity and downstream usability remain incomplete.

Possible causes

Pickup delay, scale queue, missed weighing, or missing digital record.

Example

A reel remains unweighed 34 minutes after its declaration.

Detection indicators and algorithm

Find produced-reel records with unweighed status and no scale record. Use current time minus production declaration time; warn before the deadline and alert after it.

A12 · Waste declaration

Waste bag not declared

Unsupported hypothesis

When it happens

A physical waste bag is full and closed but has no declaration or barcode record.

Why the alert exists

Waste and material balance become incomplete.

Possible causes

Operator forgot the declaration or label.

Example

A closed waste bag remains beside the machine without a digital record.

Detection indicators and algorithm

Current systems provide no independent signal proving that the physical bag exists. This cannot become a version 1 alert unless another reliable event is identified.

A13 · Waste weighing

Declared waste bag not weighed

Error

When it happens

A digitally declared waste bag has not been weighed within the permitted interval.

Why the alert exists

Waste quantity and OT reconciliation remain incomplete.

Possible causes

Pickup delay, scale delay, or missed weighing.

Example

Waste bag MR-10982 was declared 34 minutes ago but has no weight.

Detection indicators and algorithm

Start the timer at the waste declaration. Alert when no corresponding scale record exists after the configured threshold.

A14 · Downstream movement

Incomplete reel used downstream

Candidate error

When it happens

A produced reel is transferred to or consumed by another OT while required information, such as its weight, is missing.

Why the alert exists

Incomplete data propagates into another production stage.

Possible causes

Premature movement, missing weighing, or insufficient ERP validation.

Example

An unweighed printed reel is sent to lamination.

Detection indicators and algorithm

When a downstream movement or consumption event occurs, verify that the source reel has all required fields and prior events. This remains a candidate until confirmed against ERP validations.

REFERENCE

Excluded or non-primary rules

Consumed reel was not reserved for the OT

Impossible: EMUSA Soft prevents this consumption.

Production declared while an active OT has no consumption

Duplicate: the active-OT-without-consumption alert already exists.

Rewinder completion detected directly

No PLC or machine-completion signal currently exists.

Physical label was not printed

Not observable without printer acknowledgment or print-job failure data.

Scale discovers an undeclared reel

Useful as late reconciliation evidence, but too late to be the primary warning.
